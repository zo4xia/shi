# 2026-04-13 配置面板输入区布局规则记录

## 目的

给配置面板里“需要用户填入内容的区域”定一条统一布局规则。

这条规则不是单个组件私规，
而是后续所有配置面板表单块的共同约束。

## Rule 1：先判断能否一行解决

做布局前，先思考：

- 当前字段与说明，是否能在一行或一个紧凑块内解决

如果能解决：

- 不要为了“看起来整齐”就强行拉成长竖排
- 不要制造过多空行、空白列、装饰性留白

## Rule 2：看内容占比，不看表面松弛感

判断一个配置块是否合理，不是只看有没有空白，
而是看：

- 有实际内容的区域
- 与没有实际内容、只是空格/空行/被浪费的留白区域

二者是否失衡。

目标原则：

- 有内容 : 无实际内容留白
- 约为 `7 : 3`

如果空白比例已经过半：

- 说明当前块布局失衡
- 必须重排

## Rule 3：空白过半时，不继续堆竖排

如果一个块空白过多，
优先考虑以下收敛方式，而不是继续往下堆：

- 并列布局
- 递进菜单式
- 手风琴
- 伸缩窗
- 弹窗
- 分步展开

## Rule 4：优先做减法

配置面板不是展示海报，
它首先是“高密度、有工作效率的输入界面”。

因此优先级固定为：

1. 信息是否在视线范围内快速成立
2. 填写动作是否短链路
3. 空白是否被控制
4. 视觉是否整齐

也就是说：

- 先收敛结构
- 再谈装饰

## 对当前主家的含义

这条规则特别适用于：

- `Settings.tsx`
- 各类配置子面板
- 原生能力设置
- MCP / Skill / IM / 路径 / API 配置区

因为这些区域天然容易犯两个问题：

1. 字段一多就无脑纵向堆叠
2. 为了“呼吸感”制造过多空白，导致内容效率变差

## 当前结论

以后配置面板的输入区布局，先过这一关：

- 能不能一行解决
- 当前块内容占比是不是失衡
- 如果空白过半，有没有改成并列 / 递进 / 折叠 / 弹层

这条规则属于主家的“做减法规则”，
目的是让配置界面始终更像工具，
而不是逐渐膨胀成松散的说明页。
