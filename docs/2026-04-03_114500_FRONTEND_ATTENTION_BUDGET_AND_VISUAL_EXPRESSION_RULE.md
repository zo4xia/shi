# 前端注意力预算与视觉表达规则

记录时间：2026-04-03 11:45:00

## 核心原则

前端不是把信息“摆上去”就结束，而是要控制用户的注意力成本。

当前达成的规则：

1. 如果一行文字长度连整个通栏的一半都占不到，就先怀疑它不该单独占一行。
2. 能改成标签、chip、状态块、图标组合的，不继续用长句。
3. 能用视觉形式表达的，不优先用说明性文字硬撑。
4. 不要让页面靠“很多行字”制造内容感，这会快速带来视觉疲劳。
5. 页面尽量不要逼用户下拉；每多下一屏，用户耐心和注意力都会下降。
6. 首屏必须优先给主链内容、主操作和关键识别信息。
7. 中屏和移动端默认做减法，低优先级内容可隐藏、折叠、延后。

## 实施提醒

- 分类优先用标签，不优先用纯文本说明。
- 角色、来源、状态优先用 chip / badge，不优先用句子描述。
- 如果一个区块用户需要“读很久”，先想能不能改成“扫一眼就懂”。
- 如果一个模块让页面长度明显增加，就先评估是否值得放在首屏。

## 使用场景

这条规则优先适用于：

- 首页信息布局
- 侧边栏导航
- 内页头部
- 技能/插件/记录这类列表页
- 对话页面的状态条、提示条、标签条
