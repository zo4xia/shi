# 高代价踩坑边界（失忆后先看这个）

如果你现在脑子是乱的、上下文不全、或者刚醒来只剩一点记忆，先看这份。

这不是灵感笔记。  
这是拿很大代价换出来的边界。

这些边界不是“建议”。  
这些边界是：

- 反复踩穿过
- 花很多时间和 token 才确认
- 一旦回退，伤害会立刻重来

所以请先记住一句话：

**不要再轻率碰这些地方。**

---

## 1. 最重要的几条

### 1. 身份不是模型

- `agentRoleKey` 才是身份
- `modelId` 不是身份
- 模型只是当前开的车

如果你把模型当身份：

- 会串脑
- 会断连续性
- 会把同一个 agent 切成几个人

这条已经反复验证过，不能再错。

### 2. 广播板是接力板，不是全文仓库

- `identity_thread_24h` 是 24 小时广播板
- 它负责“我刚刚在做什么”
- 它不是全文聊天记录

正确顺序一直是：

1. 先看广播板
2. 再看最近 3 条正文
3. 不够再回长期记忆
4. 还不够才翻更长历史

不要：

- 把广播板当全文
- 跳过广播板
- 一上来全量灌历史

### 3. 降本不能靠捂嘴

不要再做这种事：

- 把 agent 压成一次一句
- 把多步行为改成单结果壳
- 把“不会说话了”包装成优化

正确降本方式是：

- 缓存命中
- 摘要复用
- 广播板接力
- 按需加载
- 大文件按需读

### 4. 文件问题先查入口，不先怪模型

如果用户说“它没看文件”，先查四件事：

1. 文件有没有真实落盘
2. 路径是不是对的
3. 当前回合挂了哪几片
4. 工具是不是选错了

不要第一时间说：

- 模型不会读
- 用户没传
- 文件没了

我们已经踩过：

- 分片文件存在
- 路径也对
- 但工具选错了

### 5. `CoworkRunner` 不是现役主链

请用最直白的话记住：

- `HttpSessionExecutor` 是现役主链
- `CoworkRunner` 是遗留兼容壳
- 它不是现在应该继续扩大的东西

如果你看到它还活着，不要先替它解释。

先问：

- 它是不是又在回流污染主链
- 它是不是又把 agent 压回旧语义

## 2. 哪些地方最危险

只要要动这些地方，必须非常慢：

- 连续性
- 广播板
- 上下文拼接
- 缓存
- 文件读取
- 工具路由
- 定时任务执行链
- `CoworkRunner`

这些不是普通代码。  
这些是主链。

## 3. 已经确认过的坑

### 坑 1：分片文件找不到

真实情况可能是：

- 文件切出来了
- 文件夹也在
- 路径也对
- 但保存目录漂了，或者挂载没进当前回合

### 坑 2：工具用错了

真实情况可能是：

- 附件路径是真的
- 但系统把它丢给了 `browser_observe_page`
- 所以看起来像“不会读”

不是不会读，是拿错工具。

### 坑 3：广播板数据在，但 agent 口头说看不到

这通常不是广播板没了。  
更可能是：

- 不同执行链注入位置不同
- 模型口径漂移

所以不能只听它一句“我看不到”。

要去看：

- 数据表
- helper
- 执行器
- 运行时规则

### 坑 4：定时任务通知入口看起来能用，其实没接线

真实情况可能是：

- 前端能选
- 数据库存下来了
- 但调度器根本不发

这类东西不处理，会误导用户。

## 4. 你失忆时的行动顺序

如果你现在很乱，不知道从哪下手，就照这个顺序：

1. 先看 `docs/AGENTS.md`
2. 再看 `docs/CONTINUITY_NON_REGRESSION_NOTICE_2026-03-30.md`
3. 再看 `docs/BROADCAST_BOARD_ORIGINAL_BOUNDARY_2026-03-30.md`
4. 再看最新事故说明：
   - `docs/ATTACHMENT_ROUTING_POSTMORTEM_2026-03-30.md`
   - `docs/SCHEDULER_IM_AND_BROADCAST_VISIBILITY_NOTES_2026-03-30.md`
5. 然后再动代码

如果还没做完上面这些，不要急着改。

## 4.5 记录文件怎么命名

以后新写的事故记录、修复记录、复盘记录、边界说明，文件名要带上**秒级时间戳**。

不要只写日期。

原因很简单：

- 后面会有很多份记录
- 如果只有日期，容易分不清哪份在前、哪份在后
- 一旦顺序混了，失忆后就会再次误判

以后请优先用这种格式：

- `YYYY-MM-DD_HHMMSS_主题.md`

例如：

- `2026-03-30_221530_attachment-routing-postmortem.md`

如果暂时改不了旧文件，至少从**现在开始的新文件**必须这样做。
