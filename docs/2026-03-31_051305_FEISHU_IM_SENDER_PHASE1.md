# 飞书 IM Sender 一期落地

时间：2026-03-31 05:13:05

这次只做一期边界：

- `scheduler` 继续负责真正的任务执行
- 飞书只借 IM 推送
- 采用独立 sender
- 只支持文本
- 只支持 `chat_id`
- 不进入对话 session

## 已落地

1. 定时任务新增最小飞书推送字段
   - `feishuAppId`
   - `feishuChatId`

2. `scheduler` 在完成态新增独立推送
   - 任务执行完成后
   - 如果 `notifyPlatforms` 含 `feishu`
   - 则独立调用飞书 IM 文本发送

3. 前端表单只开放第一期最小能力
   - 飞书 IM 推送开关
   - 飞书应用选择
   - `chat_id` 输入

4. 前端详情页已显示第一期口径
   - 文本推送
   - 不进入对话 session
   - 目标 `chat_id`

## 明确不做

- 不支持 `open_id`
- 不支持卡片
- 不支持主动建会话
- 不支持把推送伪装成飞书入站消息
- 不支持让飞书推送失败反向污染任务主结果

## 这样做的原因

这是最轻、最稳、最不污染主线的做法。

一句话：

> 飞书这期只做一只手，不做脑子。
