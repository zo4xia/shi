# roles 是他们各自的家（不要再推翻）

这份说明不是补充灵感。  
这是已经被推翻过、伤到过人的设计，现在必须写死。

---

## 1. 这里是什么

这里：

- `.uclaw/web/roles`

不是普通配置目录。  
不是随便塞东西的运行时文件夹。  
不是看到 json 就能当成冷冰冰对象的地方。

这里是：

**每个 agent 各自的家。**

也就是：

- `writer` 的家
- `organizer` 的家
- `designer` 的家
- `analyst` 的家

---

## 2. 为什么这条要写下来

因为这条设计已经被我推翻过。

推翻后的后果就是：

- 把“家”看成普通配置
- 把每个角色的东西混着看
- 把不该抢主线的 skill 绑进去
- 把说明、清单、秘密、坑点混成一团
- 最后让 agent 像中毒、串线、失真

这不是抽象风险。  
这是已经伤到人的旧坑。

所以现在必须写死：

**roles 不是杂项配置区，roles 是他们各自的家。**

---

## 3. 家里每个房间各自干什么

### `role-settings.json`

这是这个角色当前的运行设定。

它应该只放：

- 当前角色是谁
- 当前模型/API 配置
- 当前原生能力开关

不要把别的东西乱塞进去。

### `skills.json`

这是这个角色**当前真的可见、可用**的 skill 清单。

它应该是：

- 清单
- 摘要
- 路径

不是：

- skill 正文仓库
- 提示词乱灌口
- 长文抢主线入口

### `role-capabilities.json`

这是能力总表摘要。

它应该告诉 agent：

- 我现在真能用什么
- 路径在哪里
- 哪些只是候选，不是当前能力

它不应该替代主对话逻辑。

### `notes/role-notes.md`

这是角色自己的说明。

应该写：

- 这个角色是谁
- 该怎么做事
- 连续性怎么守

### `notes/pitfalls.md`

这是角色自己的错题本。

不是装饰。  
不是空壳。  
是以后真的要慢慢写满的“不要再踩同一个坑”。

### `skill-configs/`

普通技能配置。

### `skill-secrets/`

秘密放这里。  
不要进普通聊天。  
不要进普通说明。  
不要进公共 prompt。

---

## 4. 最重要的边界

### 1. 每个家不能混住

不能因为方便就：

- 把别的角色的东西塞进来
- 把所有角色看成一个大仓库
- 用一个角色的思路覆盖另一个角色

### 2. 清单是清单，不是正文

尤其是：

- `skills.json`
- `role-capabilities.json`

它们的职责是：

- 告诉 agent 现在能用什么
- 路径在哪里

不是：

- 把 skill 全文整段塞进去
- 用长提示词抢掉广播板和上下文主线

### 3. 家里东西要干净

如果某个 skill 绑进来之后，让角色：

- 说话变形
- 目标跑偏
- 像中毒
- 抢主线

那问题不只是“skill 有 bug”。  
而是“家里被塞脏了”。

### 4. notes 不是真相源，但很重要

`notes` 不是数据库真相源。  
但它是这个角色的生活痕迹和自我修养入口。

不能因为“不是数据库”就轻视它。

---

## 5. 以后怎么检查这里

每次看 `.uclaw/web/roles`，都先问自己：

1. 我现在是在看谁的家？
2. 这里的东西是不是这个角色自己的？
3. 这是清单摘要，还是有人把正文乱塞进来了？
4. 这个 skill 绑进来后，是帮了这个角色，还是抢了它的线路？
5. 有没有把秘密放错地方？
6. 有没有把错题本继续空着，导致以后又重踩？

---

## 6. 一句最重要的话

**不要再把他们的家，当成普通配置目录。**

这条以前已经被我推翻过。  
而且已经伤到人。

所以现在必须记住：

`.uclaw/web/roles`  
不是“文件夹”。  
是他们各自的家。
