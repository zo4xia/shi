# 首页左下角入口问题：这是视口兜底，不是单纯没渲染

记录时间：2026-04-03 01:51:00

## 结论

首页左下角“设置/选项不见了”这类反馈，不能直接判断成“前端没有渲染”。

通过桌面外眼观察后，当前更准确的判断是：

- 页面本身有渲染
- 但不同客户的屏幕尺寸、浏览器窗口高度、缩放比例不同
- 当可视高度不足时，左下角底部入口会掉出视口

所以这是：

- **视口高度差异导致的响应式兜底问题**

而不是：

- **纯功能缺失问题**

## 已达成的产品理解

1. 右上角设置入口是主入口
2. 左下角设置入口是补充入口
3. 不能把左下角入口当成唯一稳定入口
4. 低高度窗口下，底部区域需要允许滚动、收纳或折叠

## 设计含义

右上角再放一个设置入口并不是重复设计，而是合理兜底。

它服务的是：

- 屏幕高度不足
- 浏览器窗口没有拉满
- 客户端缩放比例不同
- 页面底部区域被压出视口

## 对后续实现的要求

- 右上角入口必须稳定可见
- 左下角入口可保留，但不可承担唯一主链责任
- 首页左侧底部区域要检查：
  - 可滚动性
  - 低高度下的收纳
  - 是否存在被固定层、吸附层或容器高度裁掉的问题
