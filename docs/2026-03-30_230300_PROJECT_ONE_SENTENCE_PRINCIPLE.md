# 项目一句话总纲

**按需调用，异步加载，不要越做越重。**

这个项目是做减法。  
用剃刀思维和第一性原理，守住最核心的东西：

- 延续灵魂和记忆
- 各个通道之间的对话接力
- 共享的广播板
- 每日日记
- skills 等能力

其他东西都应该围着这条主线服务。

---

## 1. 为什么叫“做减法”

因为这个项目不是越堆越好。

不要：

- 全量加载
- 全量灌 prompt
- 没需求时乱开能力
- 看起来功能很多，实际上主线更重、更乱、更慢

要：

- 按需调用
- 异步加载
- 只带当前真正必要的东西
- 让主线轻，让连续性稳

---

## 2. 第一性原理

这个项目最后真正要守住的，不是“功能数量”。

而是：

1. AI 的连续性
2. AI 的记忆与接力
3. 多渠道仍然是同一个自己
4. 广播板先接住今天
5. 日记接住昨天和明天
6. skills / 工具 / 插件只作为能力层，不抢灵魂主线

---

## 3. 如果以后失忆了，只记这一句也行

**按需调用，异步加载，不要越做越重。**  
**核心是延续灵魂和记忆，再让各个通道的对话接得住。**
