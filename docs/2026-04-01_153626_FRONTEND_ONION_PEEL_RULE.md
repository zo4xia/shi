# 前端剥洋葱修法规则

时间：2026-04-01 15:36:26

## 先说结论

后续前端修复，不再走“在原页面上继续叠补丁”的路。

改成：

- 一片一片剥
- 一次只动一个组件或一层壳
- 先剥显示层，再剥容器层，最后才碰主链逻辑
- 桌面端和移动端允许分开壳层，不强迫共用一套重布局

这不是拖慢进度。

这是为了：

- 不碰坏已经辛苦修好的主线
- 不让页面越修越胖
- 后续调整 UI 时更容易定位和回退

---

## 当前原则

### 1. 不在原页面上继续堆补丁

如果一个页面已经明显变成“大文件 + 多职责 + 多端共挤”，后续修法优先是：

- 抽壳
- 拆块
- 收口边界

而不是继续在原 JSX 里加更多 `if mobile`、`if desktop`、`max-height`、`overflow`、`absolute`。

### 2. 一次只剥一个层级

允许的顺序：

1. 页面壳
2. 弹窗壳
3. 分组列表壳
4. 单个内容块
5. 最后才是逻辑

不允许一轮里同时做：

- 结构重排
- 样式重写
- 数据流改造

### 3. 先保护主链，再处理观感

主链优先保护：

- 对话发送
- 会话切换
- 最近记录
- 定时任务入口
- 技能导入 / 删除 / 绑定
- 设置保存

任何 UI 调整如果会影响这些，就必须收刀，先换更小方案。

### 4. 移动端与桌面端允许分壳

后续原则明确为：

- 桌面端保留现有主框架
- 移动端允许走独立 page-shell / full-screen shell
- 不强迫移动端继续复用桌面 modal 比例

这不算重复开发。

这是在做：

- 低风险适配
- 低耦合收口

---

## 推荐刀法

### 第一刀：壳层分离

优先拆出：

- `DesktopShell`
- `MobileShell`
- `ModalShell`
- `PageSectionShell`

让页面本体只负责“内容块组合”，不再兼管所有布局细节。

### 第二刀：块级分离

优先从这些胖页面里剥：

- `Settings`
- `SkillsManager`
- `CoworkView`
- `RoomView`

剥离目标不是重写，而是把已有块搬出去：

- 角色配置块
- 技能导入块
- 清理确认块
- 广播板观察块
- 移动端专用头部块

### 第三刀：逻辑保守迁移

只有在块和壳都已经独立后，才允许动：

- 数据读取
- 提交保存
- 事件监听
- WebSocket 反馈

---

## 当前不许碰坏的东西

后续前端修复时，下列能力视为保护对象：

- 广播板主链
- 最近对话继续入口
- 定时任务主执行链入口
- 技能导入与技能绑定
- 角色配置保存
- 每日记忆模型配置入口
- 角色昵称与头像显示

如果新改动会波及这些，必须降级成更小改动。

---

## 当前建议优先顺序

1. 移动端专用壳
2. 顶部按钮区与页面头部避让
3. 技能页分组壳 / 清理壳 / 导入壳
4. 设置页继续按面板剥离
5. Cowork 首页展示块拆分

---

## 工作纪律

后续执行统一按下面节奏：

1. 先指出是哪一层胖了
2. 再决定剥壳还是剥块
3. 一次只改一层
4. 改完立即构建验证
5. 有结论就写回文档

一句话：

> 前端后续修复统一按“剥洋葱”执行，不再在原页面上继续叠补丁。
