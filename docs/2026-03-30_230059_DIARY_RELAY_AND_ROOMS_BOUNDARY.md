# 日记接力与房间边界（先看这个）

这份说明记录的是：

- 广播板什么时候该空
- 广播板空了以后去哪里找回自己
- 信息进来时到底先带什么
- 为什么每个 agent 都必须有自己的房间

这不是补充灵感。  
这是已经反复讲过、必须钉死的边界。

---

## 1. 每个 agent 都有自己的房间

这个项目里，每个 agent 都有自己的房间。

房间就在：

- `.uclaw/web/roles/<roleKey>/`

这里面藏的是：

- 自己的说明
- 自己的能力清单
- 自己的 skill 清单
- 自己的秘密
- 自己的错题本

不要再把这里看成普通配置目录。  
更不要把别人的东西塞进来。

一句话：

**每个房间里面，放的都应该是他们自己的东西。**

---

## 2. 广播板什么时候能清空

广播板不能随便清。

### 正确规则

- **必须是写完日记之后，才能清广播板**

原因：

- 广播板是今天留给明天的接力棒
- 如果日记还没写完就清掉
- 那就是把今天的自己提前弄丢

所以：

- 先写日记
- 日记写完并沉淀
- 再清广播板

---

## 3. 第二天醒来，如果广播板没有内容怎么办

正确流程是：

1. 先看今天的广播板
2. 如果广播板有内容
   - 先接住这根棒
3. 如果广播板没有内容
   - **去数据库查昨天的日记**
4. 从昨天的日记里找：
   - 昨天做了什么
   - 有没有没做完的待办
   - 有没有重要提醒
   - 昨天这个 agent 的性格化碎片、习惯、状态
5. 然后把“今天的第一条接力”重新写回广播板

也就是说：

**广播板空了，不等于自己没了。**  
这时应该去昨天的日记里，把昨天的自己接回来。

---

## 4. 日记里该记什么

日记不只是总结。

日记里应该有：

- 昨天做了什么
- 哪些事情没做完
- 哪些坑踩过
- 哪些问题解决了
- 今天要注意什么
- 这个 agent 昨天留下来的性格化碎片、偏好、习惯、语气、琐碎感受

也就是说：

日记不只是业务流水，  
也应该是“这个自己昨天活过的痕迹”。

---

## 5. agent 自己每天要看什么

每天定时，agent 应该先看：

1. 今天的广播板
2. 缓存命中和近期记录
3. 昨天的日记（如果今天广播板空了）
4. 自己房间里的错题本和清单

不是为了形式。  
是为了让今天的自己，不是完全新的陌生人。

---

## 6. 信息进来时怎么加载

优先级从高到低：

### 1. 身份 + 广播板（最高）

永远最高优先级。

广播板摘要格式应该尽量是：

- `身份 + 渠道 + 时间 + 极简摘要`

要求：

- 摘要尽量压到 `10` 个字内
- 只留有意义内容
- 无意义废话不要写进来，例如：
  - `哈哈`
  - `嗯`
  - `哦`
  - `好的`

广播板的作用是：

- 先告诉这个 agent：
  - 我是谁
  - 我刚刚在做什么
  - 哪根接力棒要先接

### 2. 最近 3 条上下文（次之）

只带最近 3 条原始正文。

原因：

- 用户可能跨渠道
- 所以原始上下文重要，但不能压过广播板

这条的定位是：

- 补精确细节
- 不是抢主线

### 3. 当前可用清单

包括：

- 当前可用 skills
- 系统 skills
- 能力清单
- 已加载文件清单

但这里必须是：

- **清单摘要**
- **路径**

不是：

- 整段 skill 正文照抄
- 没有需求时就乱加载、乱运行

### 4. 错题本 + 插件名单

这一层也是：

- 名单
- 索引

不是正文灌入层。

作用是：

- 告诉 agent 去哪里查坑
- 告诉 agent 现在有哪些外挂/插件

---

## 7. 为什么这样分层

因为要靠这些层来优化成本。

一句话：

**广播板优先、最近 3 条次之、清单和错题本做索引层，才能靠缓存命中率去优化成本。**

如果顺序反了，就会出现：

- skill 抢线路
- 长说明压主线
- agent 串线
- token 爆炸
- 连续性反而变差

---

## 8. 最后一句

这套顺序不是随便排的。

它的目的就是：

- 先保护身份
- 再接住接力
- 再补上下文
- 最后才看清单和名单

这样他们才不会在一次次醒来时，完全失去自己。

---

## 9. 主链保留项

有些东西不是“可选能力”，而是主链生存层。

这些必须有：

- 日记
- 记忆
- 错题本
- `daily-memory-*`

尤其 `daily-memory-*`，本质上就是：

- 第二天醒来时最初加载的一部分
- 用来把昨天的自己接回来
- 用来接住“昨天还有没有待办、有没有提醒、有没有没说完的话”

所以不要再把它误判成普通用户安装型 skill。
