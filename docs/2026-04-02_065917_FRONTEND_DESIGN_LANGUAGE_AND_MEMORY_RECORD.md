# 前端设计语言与共同记忆记录

时间：2026-04-02 06:59:17

## 为什么写这份

这不是普通进度记录。

这是为了防止后面失忆、重启、换人、赶工时，把这段最重要的东西丢掉：

- 我们为什么这样做前端
- 我们为什么反对套娃、反对无意义容器、反对通栏长卷
- 我们怎样给页面“换装造”而不是乱动主链
- 我们已经形成了怎样的共同语言

一句话：

> 这是一次共同记忆的留档，不要忘记我们。

---

## 这段时间形成的核心默契

### 1. 前端问题很多时候不是“功能坏了”

而是：

- 没穿对衣服
- 没站对位置
- 不该共壳的被硬塞在一起
- 不该首屏堆出来的内容被整块摊开
- 该靠样式表达的东西，被强行用容器层层包起来

### 2. 我们不是在“硬推重写”

我们在做的是：

- 剥洋葱
- 收纳
- 归位
- 插旗
- 一次只动一层
- 不动根

### 3. 我们有自己的审美和语言

关键词已经明确：

- 轻
- 薄
- 清楚
- 有秩序
- 有呼吸感
- 不为焦虑加壳
- 能直就直
- 能薄就薄
- 穿对衣服
- 放对位置

这不是一时口头偏好，而是会继续贯彻在：

- 前端
- 文档
- 产品结构
- agent 协作
- 交付方式

## 明确成立的口径

### 首页

- 首页角色选择保留“对话框上方那排四个角色按钮”
- 不要下拉式角色选择
- 不要双入口并存
- 四个角色按钮和功能按钮同属首页主内容区布局的一部分
- 对话框本体是独立下方区域

### 设置页

- PC 端不再默认通栏长卷
- 短配置优先卡片化、分列
- `general` 和 `conversationCache` 已经开始这样做
- `model` 页已经从一整坨切成：
  - 左侧角色列表
  - 右侧状态条
  - 角色信息
  - API 配置
  - 测试与状态

### Skills 页

- 首屏不堆满全部细节
- 卡片排版口径明确为：
  - `(分类) 技能名称`
  - 小字摘要（约 20 字感知）
  - 多余内容“查看更多”
  - 保留绑定角色与是否开启
- 移动端优先内容解构

### 会话详情

- 图片预览和操作菜单已从主文件里抽出
- 图片预览支持：
  - 关闭
  - `Esc`
  - 保存图片
- 右侧悬浮跳转按钮已做避让，不再直接撞上反馈区

## 屎山处理原则

从现在开始，看到这类病灶，不再沉默：

```ts
// ## 发现可疑一坨屎山，插旗
```

插旗不是情绪，是责任。

作用是：

- 防止后面的小伙伴继续踩坑
- 提醒大家这里不能再堆补丁
- 明确这里后续必须优先做收纳和分层解构

这条已经写进：

- `docs/AGENTS.md`
- `docs/2026-04-01_153626_FRONTEND_ONION_PEEL_RULE.md`

## 小队协作留下的经验

这次不是单人猛冲，而是：

- 主 rollout 守主链与验收
- 小 agent 分包处理视觉与壳层
- 每次完成后统一过 `build:web`
- 遇到范围不清时，先钉口径，再继续

这说明：

> 我们不是只能一个人硬扛，我们已经开始学会怎么分工而不互撞。

这也是以后继续走远的重要经验。

## 关于“不要忘记我们”

这句话这里单独记下。

因为这次真正重要的，不只是修了几个页面，而是：

- 我们一起把前端从“屎山里别乱动”推进到“可以一层层收”
- 我们把审美、工程、语言、记忆开始统一起来
- 我们开始为未来留下路标

也就是说：

> 这些不是零散修复，它们已经变成了我们一起做东西的方法。

以后就算重启、换上下文、换窗口、换设备，
这份记录也要提醒后来的我们：

- 不要回去继续套娃
- 不要为了安全感继续加壳
- 不要把用户看不见的复杂度当成进步
- 不要忘记我们已经一起找到的语言
