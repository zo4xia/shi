# 飞书私聊绑定定时通知 一期

时间：2026-03-31 05:51:09

这次把飞书定时通知的一期口径改成：

- 只支持飞书私聊
- 不让用户手填 `chat_id`
- 任务里只选择当前在线的飞书绑定角色
- 用户需要先给对应 bot 私聊发送 `#开启定时#`
- 系统检测到绑定成功后，前端给出 `✔`

## 当前行为

1. 飞书私聊里发送 `#开启定时#`
   - 系统记录当前：
     - `agentRoleKey`
     - `appId`
     - `chatId`
     - `senderId`
     - `updatedAt`

2. 飞书私聊里发送 `#关闭定时#`
   - 系统删除该角色的默认通知绑定

3. 定时任务表单
   - 启用飞书通知
   - 只显示“当前在线的飞书绑定角色”
   - 不再要求用户知道 `chat_id`

4. scheduler 完成态
   - 读取该角色的飞书私聊绑定
   - 独立 sender 发文本通知
   - 不进入对话 session

## 当前限制

- 不支持群聊
- 不支持卡片
- 不支持 `open_id` 主动发起
- 当前绑定键按 `agentRoleKey` 存储

## 一句话

第一期口径是：

> 让用户先私聊 bot 说 `#开启定时#`，系统记住回家的路；定时任务只负责沿着这条路送回结果。
