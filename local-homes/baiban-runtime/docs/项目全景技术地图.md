## 二、数据结构全景图

### 2.1 核心类型定义

```typescript
// ==========================================
// 📦 Script 剧本结构（后端生成，前端消费）
// ==========================================
interface Script {
  problemId: string           // 唯一标识
  problem: string             // 原始题目文本
  problemType: 'text' | 'graph'  // 题目类型：纯文本/画图题
  audio: {                    // 音频数据
    data: string              // Base64编码的WAV音频
    duration: number          // 音频时长（秒）
    text: string              // 完整讲解文本
  }
  cues: Cue[]                 // 时间轴打点数组
  layout: {                   // Canvas布局配置
    textArea: { x: number; y: number; width: number; height: number }
    graphArea?: { x: number; y: number; width: number; height: number }
  }
}

// ==========================================
// 📦 Cue 打点结构（时间轴事件）
// ==========================================
interface Cue {
  id: string                  // 打点ID（序号）
  time: number                // 触发时间（秒）
  type: 'write' | 'draw' | 'pause'  // 动作类型
  boardItem: BoardItem        // 板书内容
  note?: string               // 讲解备注
}

// ==========================================
// 📦 BoardItem 板书项
// ==========================================
interface BoardItem {
  type: 'text' | 'graph'      // 内容类型
  content: string             // 文本内容或描述
  position: { x: number; y: number }  // Canvas坐标
  graphType?: 'coordinate' | 'function' | 'geometry' | 'point' | 'line'
  graphData?: {
    function?: string         // 函数表达式，如 "2*x+1"
    points?: { x: number; y: number; label?: string }[]
    shapes?: { type: string; data: any }[]
    range?: { xMin: number; xMax: number; yMin: number; yMax: number }
  }
}
```

## 五、WebSocket通信协议

### 5.1 事件清单

| 方向 | 事件名 | 数据结构 | 说明 |
|------|--------|----------|------|
| 前端→后端 | `generate-script` | `{ problem: string }` | 请求生成讲解剧本 |
| 前端→后端 | `configure-api` | `{ apiKey: string, baseUrl?: string }` | 配置API密钥 |
| 前端→后端 | `play` | `{ sessionId: string }` | 开始播放 |
| 前端→后端 | `pause` | `{ sessionId: string }` | 暂停播放 |
| 前端→后端 | `stop` | `{ sessionId: string }` | 停止播放 |
| 后端→前端 | `status` | `{ stage: string, message: string }` | 状态更新 |
| 后端→前端 | `script-ready` | `{ sessionId: string, script: Script }` | 剧本生成完成 |
| 后端→前端 | `error` | `{ message: string }` | 错误信息 |
| 后端→前端 | `api-configured` | `{ success: boolean }` | API配置成功 |

1. **三Agent协作模型**：导演-板书员-播音员分离，各司其职
