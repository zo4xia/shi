# 智能手写教学白板系统 - 技术方案文档

## 一、项目概述

### 1.1 项目背景
传统教学视频制作需要教师手动录制板书讲解，存在以下痛点：
- 录制成本高：需要专业设备和后期剪辑
- 内容固化：已录制视频难以修改
- 个性化不足：无法根据学生需求调整讲解方式
- 效率低下：一道题需要反复录制多遍

### 1.2 解决方案
开发一套**智能手写教学白板系统**，通过AI自动生成解题步骤和讲解语音，Canvas实现逐笔手写动画，语音和板书实时同步播放。

## 三、核心设计

### 3.1 三Agent协作模型

本系统的核心创新点在于**三Agent协作模型**，模拟真实教学场景中的三个角色：

```
┌──────────────────────────────────────────────────────────────┐
│                      Agent C (导演)                           │
│  职责：时间轴监控、打点触发、节奏控制                           │
│  ────────────────────────────────────────────────            │
│  • 监听音频播放进度                                           │
│  • 根据时间轴触发板书/画图事件                                 │
│  • 协调Agent A和Agent B的配合                                 │
└──────────────────────────────────────────────────────────────┘
           │                                    │
           │ 触发事件                           │ 同步进度
           ▼                                    ▼
┌──────────────────────┐            ┌──────────────────────┐
│   Agent A (板书员)    │            │   Agent B (播音员)    │
│  职责：Canvas逐笔手写  │            │  职责：语音播放       │
│  ──────────────────   │            │  ──────────────────  │
│  • 文字逐笔书写动画    │            │  • 音频播放控制      │
│  • 手绘坐标系/函数图   │            │  • 进度回调通知      │
│  • 手绘风格渲染       │            │  • 播放状态管理      │
└──────────────────────┘            └──────────────────────┘
```

**设计理由：**
1. **关注点分离**：每个Agent只负责一件事，便于维护和扩展
2. **状态可观测**：各Agent状态独立，便于调试和展示
3. **灵活协作**：通过事件驱动，可实现复杂的协作逻辑

### 3.2 时间轴同步方案

**核心挑战**：如何让板书和语音同步？

**方案演进**：
```
方案A（不可行）: 板书写完 → 计算时长 → 生成语音
❌ 问题：板书速度不固定，无法精确预测

方案B（不可行）: 强制板书速度匹配语音时长
❌ 问题：书写速度不自然，体验差

方案C（采用）: 预生成语音 → 计算打点时间 → 触发板书
✅ 优势：语音节奏自然，板书自由书写
```

**实现原理**：
```typescript
// 1. LLM生成讲解剧本，包含多个片段
segments: [
  { type: "intro", speak: "这道题考查...", boardItems: [] },
  { type: "main", speak: "我们来看步骤...", boardItems: [...] },
  { type: "summary", speak: "总结一下...", boardItems: [] }
]

// 2. TTS生成语音，获取精确时长
const audio = await TTS(fullText);
const duration = audio.duration; // 精确的音频时长

// 3. 计算每个板书项的触发时间点
const cues = calculateCueTimes(segments, duration);
// 结果示例：
// cue[0]: { time: 0.5, content: "解：" }
// cue[1]: { time: 2.3, content: "设x为..." }
// cue[2]: { time: 5.0, type: "draw", graphType: "coordinate" }

// 4. Agent C实时监控音频进度，触发对应板书
setInterval(() => {
  const currentTime = audio.currentTime;
  if (currentTime >= cue.time) {
    triggerBoardWork(cue);
  }
}, 100);
```

**同步原则**：
> **"大差不差，内容对应即可"** —— 不强求精确对齐，保证内容逻辑正确即可

### 3.4 手绘风格实现

**设计目标**：让板书看起来像真人手写，而非机械打印。

**实现技术**：
1. **线条抖动**：随机偏移使线条不完美
2. **重复描边**：模拟手写时的重复描画
3. **压感模拟**：笔画粗细随位置变化
4. **自然曲线**：使用贝塞尔曲线插值

```typescript
// 手绘线条核心算法
drawLine(x1, y1, x2, y2) {
  // 1. 添加随机抖动
  const rough = 2;
  const startX = x1 + (Math.random() - 0.5) * rough;
  const startY = y1 + (Math.random() - 0.5) * rough;
  
  // 2. 使用贝塞尔曲线使线条更自然
  const midX = (x1 + x2) / 2 + (Math.random() - 0.5) * rough;
  const midY = (y1 + y2) / 2 + (Math.random() - 0.5) * rough;
  
  ctx.quadraticCurveTo(midX, midY, endX, endY);
  
  // 3. 30%概率画第二条线（模拟重复描边）
  if (Math.random() > 0.7) {
    ctx.globalAlpha = 0.3;
    // 再画一遍...
  }
}
```

## 四、核心功能模块

### 4.1 剧本生成模块

```typescript
async function generateScript(problem: string): Promise<Script> {
  // Step 1: LLM分析题目类型并生成讲解剧本
  const response = await LLM.chat({
    system: `你是一位资深数学老师...`,
    user: `题目：${problem}`
  });
  
  // 解析JSON格式的剧本
  const script = JSON.parse(response);
  
  // Step 2: 拼接完整讲解文本
  const fullText = script.segments.map(s => s.speak).join(' ');
  
  // Step 3: TTS生成语音
  const audio = await TTS.create({
    text: fullText,
    voice: 'tongtong',
    speed: 0.9
  });
  
  // Step 4: 计算板书打点时间
  const cues = calculateCues(script.segments, audio.duration);
  
  return { script, audio, cues };
}
```

### 4.2 实时同步播放模块

```typescript
class AgentC_Director {
  private cues: Cue[] = [];
  private currentIndex: number = 0;
  
  start(getCurrentTime: () => number) {
    // 每100ms检查一次
    setInterval(() => {
      const time = getCurrentTime();
      
      // 触发所有已到时间的打点
      while (this.currentIndex < this.cues.length 
             && time >= this.cues[this.currentIndex].time) {
        this.triggerCue(this.cues[this.currentIndex]);
        this.currentIndex++;
      }
    }, 100);
  }
  
  triggerCue(cue: Cue) {
    if (cue.type === 'write') {
      AgentA.writeText(cue.content, cue.position);
    } else if (cue.type === 'draw') {
      AgentA.drawGraph(cue.graphType, cue.graphData);
    }
  }
}
```

## 五、API配置方案

### 5.1 为什么需要动态配置？

- **安全性**：API Key不应硬编码在前端
- **灵活性**：支持多环境部署（开发/测试/生产）
- **可维护性**：无需修改代码即可更换配置

### 5.2 配置流程

```
用户 → 设置界面 → 输入API Key/Base URL → WebSocket发送配置
                                                    ↓
后端存储配置 ← 验证配置有效性 ← 接收配置
     ↓
后续请求使用该配置调用AI服务
```

### 5.3 支持的配置项

| 配置项 | 必填 | 说明 |
|--------|------|------|
| API Key | 是 | AI服务认证密钥 |
| Base URL | 否 | 自定义API地址，默认使用官方地址 |
